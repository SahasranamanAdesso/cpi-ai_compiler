### 2.3 BPMN XML Structure (.iflw)

#### Root Element
```xml
<bpmn2:definitions 
    xmlns:bpmn2="http://www.omg.org/spec/BPMN/20100524/MODEL"
    xmlns:bpmndi="http://www.omg.org/spec/BPMN/20100524/DI"
    xmlns:dc="http://www.omg.org/spec/DD/20100524/DC"
    xmlns:di="http://www.omg.org/spec/DD/20100524/DI"
    xmlns:ifl="http:///com.sap.ifl.model/Ifl.xsd"    ← SAP EXTENSION
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    id="Definitions_1">
```

**6 Namespaces Required:**
1. `bpmn2` - Core BPMN 2.0
2. `bpmndi` - BPMN Diagram Interchange
3. `dc` - OMG DD (Diagram Definition) - Bounds
4. `di` - OMG DD - Waypoints
5. `ifl` - **SAP Integration Flow Extensions**
6. `xsi` - XML Schema Instance

#### Hierarchy

```
<bpmn2:definitions>
  ├── <bpmn2:collaboration>           # Container for participants
  │   ├── <bpmn2:extensionElements>
  │   │   └── <ifl:property>*         # 14 collaboration properties
  │   ├── <bpmn2:participant>*        # Sender/Receiver adapters
  │   │   └── <bpmn2:extensionElements>
  │   │       └── <ifl:property>*
  │   └── <bpmn2:messageFlow>*        # Adapter connections
  │       └── <bpmn2:extensionElements>
  │           └── <ifl:property>*     # 20+ adapter properties
  │
  ├── <bpmn2:process>                 # Integration logic
  │   ├── <bpmn2:extensionElements>
  │   │   └── <ifl:property>*         # 4 process properties
  │   ├── <bpmn2:startEvent>
  │   ├── <bpmn2:endEvent>
  │   ├── <bpmn2:callActivity>*       # Integration steps
  │   └── <bpmn2:sequenceFlow>*       # Connections
  │

```

### 2.4 SAP Extension Properties (`<ifl:property>`)

**Format:**
```xml
<ifl:property>
    <key>propertyName</key>
    <value>propertyValue</value>
</ifl:property>
```

**Collaboration-Level (14 properties):**
- `namespaceMapping`, `httpSessionHandling`, `accessControlMaxAge`
- `returnExceptionToSender`, `log`, `corsEnabled`
- `exposedHeaders`, `componentVersion`, `allowedHeaderList`
- `ServerTrace`, `allowedOrigins`, `accessControlAllowCredentials`
- `allowedHeaders`, `allowedMethods`, `cmdVariantUri`

**Participant Properties:**
- `enableBasicAuthentication`
- `ifl:type` (e.g., "EndpointSender")

**MessageFlow (Adapter) Properties (20+):**

- `TransportProtocol`, `MessageProtocol`

**Process Properties:**

**CallActivity Properties:**

### 2.5 Element Types Observed

| BPMN Element | SAP Usage | Example |
|--------------|-----------|---------|

```
│       └── content-modifier.iflw
├── unit/                   # Unit tests (current)
└── integration/            # Integration tests
    └── golden.test.ts
```

### Test Strategy

**NOT simple string comparison** - BPMN allows flexibility

**Semantic Comparison:**
1. Parse both XMLs to DOM
2. Normalize (remove whitespace, sort attributes)
3. Compare structure:
   - Same elements?
   - Same attributes?
   - Same properties?
   - Same connections?
4. Ignore:
   - Whitespace differences
   - Attribute ordering
   - Element ordering (within constraints)
   - ID numbering (StartEvent_2 vs StartEvent_1)
   - Diagram coordinates (can differ)

**Implementation:**
```typescript
function compareIFLW(expected: string, generated: string): Diff[] {
    const exp = parseXML(expected);
    const gen = parseXML(generated);
    
    return [
        ...compareDefinitions(exp, gen),
        ...compareCollaboration(exp, gen),
        ...compareProcess(exp, gen),
        // Diagram optional for tests
    ];
}
```

### Test Cases (Priority)

1. **Minimal** - Start → End only
2. **Single Component** - Start → ContentModifier → End
3. **Agg Test** - Match exported artifact exactly
4. **Linear Flow** - Multiple steps, no branching
5. **Router** - Conditional branches

## 14. Final Answer

## Is this repository ready for implementing the Compiler Backend?

### **NO** ❌

---

### Why Not?

**1. Critical IR Incompleteness (60% Gap)**

The Intermediate Representation is missing **6 major concepts** required for SAP Integration Suite:

❌ `BpmnCollaboration` - Container for participants  
❌ `BpmnParticipant` - Sender/Receiver adapters  
❌ `BpmnMessageFlow` - Adapter connections  
❌ `IflProperty` - SAP extension properties (used EVERYWHERE)  
❌ `BpmnDefinitions` - Root structure with namespaces  
❌ `BpmnDiagram` - Visual layout (MANDATORY, not optional)

**Current IR can represent:**
- Start/End events
- Components (callActivity)
- Sequence flows

**Current IR CANNOT represent:**
- HTTPS adapters (no participants)
- Adapter configuration (no message flows)
- ANY SAP-specific metadata (no extension properties)
- Visual layout (no diagram)
- Proper BPMN structure (no collaboration/definitions)

**Building backend now = guaranteed failure on import to CPI**

---

**2. Real SAP Artifacts Are FAR More Complex**

**What we thought:**
```xml
<bpmn2:process>
  <bpmn2:callActivity name="Content Modifier"/>
</bpmn2:process>
```

**What SAP actually requires:**
```xml
<bpmn2:definitions> (6 namespaces)
  <bpmn2:collaboration> (14 properties)
    <bpmn2:participant> (Sender - 2 properties)
    <bpmn2:participant> (Process)
    <bpmn2:messageFlow> (HTTPS - 20 properties)
  </bpmn2:collaboration>
  <bpmn2:process> (4 properties)
    <bpmn2:startEvent> (2 properties)
    <bpmn2:callActivity> (8 properties)
    <bpmn2:endEvent> (2 properties)
    <bpmn2:sequenceFlow>
  </bpmn2:process>
  <bpmndi:BPMNDiagram>
    <bpmndi:BPMNPlane>
      <bpmndi:BPMNShape> (with coordinates)
      ...
    </bpmndi:BPMNPlane>
  </bpmndi:BPMNDiagram>
</bpmn2:definitions>
```

**SAP Integration Suite is 10x more complex than generic BPMN.**

Extension properties (`<ifl:property>`) appear in:
- Collaboration (14 properties)
- Every participant (2+ properties each)
- Every message flow (20+ properties each)
- Process (4 properties)
- Every start/end event (2 properties each)
- Every call activity (6-15 properties each)

**We cannot hardcode these - we need a property system.**

**Problem:** Building backend now means choosing which to extend.

**Risk:** Accidentally building on the wrong foundation.

**Solution:** Archive old code first, make the path forward obvious.

---

**4. No Validation Infrastructure**

**Cannot answer:** "Is our XML correct?"

**Need:**
- Golden test framework

- Real CPI import testing

**Building without validation = flying blind**
